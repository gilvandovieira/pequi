What's inside

A logger that follows Pino's API shape — and is honest about the rest.

The TypeScript layer owns argument normalization, serializers, redaction, formatting, and JSON encoding. Every behavior below is checked against a pinned pino@10.3.1 oracle.

{ }

Pino-shaped API

Factory forms, level methods, child loggers, bindings, level-change events, formatters, mixins, and hooks.logMethod.

⌘

Safe encoder

Circular refs become "[Circular]", BigInt stays numeric, Dates keep their ISO form. Bare JSON.stringify throws; Pequi doesn't.

±

Custom levels

customLevels, useOnlyCustomLevels, and ASC/DESC/custom levelComparison — generated per-level methods, inherited by children.

✳

Real redaction

Wildcards (a.*, *.secret), array & bracket paths (a[*].b, a["x.y"]), remove, and array-path censors.

⇉

multistream & onChild

Fan out to many destinations with per-stream levels and dedupe; a root onChild that fires for every descendant.

⚙

Optional Rust sink

Pure TypeScript by default; native: "auto" loads a Rust write/flush backend over FFI when it helps. The API never changes.

→ {"level":30,"big":10}

Insertion order is preserved at every level to match Pino's line output; toJSON, non-finite numbers, and dropped undefined/function/symbol values all follow JSON.stringify semantics.

Findings

Two experiments, honestly reported.

Rolldown bundling: a marginal win

Scope-hoisting the module graph buys roughly +3–7% on average — but the per-case noise floor (~15% CV) is larger than the effect, so it's a small, real, broadly-positive change, not a step function. Minification adds nothing reliable. Output is verified byte-identical to source. Ship the readable bundle; skip minification.

Rust native: only for real I/O

For these discard/memory micro-benchmarks the native FFI sink is ~45% slower — the per-write FFI crossing costs more than it saves. It reaches parity only on file bursts. Native pays off for sustained buffered I/O, not per-line micro-loops — so it's opt-in, never the default.

Full methodology and per-case tables live in the repository at reports/rolldown-bundle-investigation.md.

Pequi is a Deno-first structured logger, planned for publication as @pequi/log. Benchmarks are micro-benchmarks on a single machine and V8 build; only relative deltas are meaningful, and the ~15% noise floor means individual rows are indicative, not definitive.
